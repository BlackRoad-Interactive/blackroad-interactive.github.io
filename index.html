Architecture
Entity-Component-System
The game engine uses ECS — entities are IDs, components are data, systems are logic. No inheritance hierarchies, no God objects. Add physics to anything by attaching a RigidBody component. Add rendering by attaching a Sprite component.
Entity = ID (uint64)
Component = Data (Position, Velocity, Sprite, RigidBody)
System = Logic (PhysicsSystem, RenderSystem, AudioSystem)

game_engine.py → creates entities, manages scenes
physics.py → PhysicsSystem queries (Position, RigidBody)
renderer.py → RenderSystem queries (Position, Sprite)
audio_engine.py → AudioSystem queries (Position, AudioSource)

Language
RoadC — game scripting language
Syntax
space GameWorld:
  let gravity = 9.81
  var player_count = 0

  fun spawn_entity(name, x, y):
    let entity = create(name)
    entity.position = (x, y)
    player_count = player_count + 1

  fun update(dt):
    for entity in entities:
      match entity.state:
        "idle" → entity.think(dt)
        "moving" → entity.physics(dt)
